Initially the entry was tested on Ubuntu 22.04 and 24.04 with curl versions 7.81.0 and 8.5.0 respectively. The segmentation fault occured within dash (when launching with sh) and bash within `__GI__libc_read()`. q3w3e3's version was curl 8.13.0.

The output on both curl 7.81.0 and 8.5.0
```
▶ sh curl\ -L\ binary.golf\ --request-target\ 5%2f5\ -O 
curl: Remote file name has no length
curl: (23) Failed writing received data to disk/application
Segmentation fault (core dumped)
```

There were some changes to the code outlined here:
- Original Issue: **curl -JO [hostname] with no Content-Disposition header fails** https://github.com/curl/curl/issues/14939 
- The behavior was changed in this PR titled **tool_cb_wrt: use "curl_response" if no file name in URL** https://github.com/curl/curl/pull/14940
- Changelog for version 8.10.1 https://curl.se/ch/8.10.1.html 

BusesCanFly was able to verify that this worked as described
